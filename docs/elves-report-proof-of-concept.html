Proof of concept: worker report to manager

Elves Report

A night-shift report for the human who wakes up to a changed branch. It is not a generic dashboard. It is a workbench ledger: what the workers accomplished, what they discovered, and what the manager should inspect next.

after the final review before the manager merges collapsible batches

Morning note for the manager

The workers should not hand back a pile of raw logs. They should leave a report that makes the night's work inspectable: the status at dawn, the problems found, the lessons worth preserving, and the exact proof behind every claim.

  • 1Status first. Branch, PR, head SHA, checks, and whether stopping is actually allowed.
  • 2Findings before flourish. Real bugs, review blockers, UX gaps, repeated failures, and caveats.
  • 3Expandable batch ledger. Skim all batches, then open the ones that need a closer look.

What belongs in the report

Not a victory lap. A ledger.

The best report makes the human feel oriented and in control before they ever open the PR diff.

01

Problems found

Bugs, broken links, auth risks, race conditions, stale docs, recurring review themes.

02

Lessons learned

Durable findings promoted from the night's work into future agent memory.

03

Proof

Commands, tests, browser checks, review loops, PR status, and known non-fatal warnings.

04

Next moves

What the manager should review, merge, deploy, rerun, or plan after breakfast.

Reusable starting point

Workers get a template.

Real Elves Reports should start from references/elves-report-template.html, then adapt the content and visual tokens to the project they worked in. The template preserves the key reporting structure without forcing this exact visual style onto every repo.
