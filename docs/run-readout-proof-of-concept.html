Elves Run Readout

A static HTML morning briefing generated at the end of a substantial Elves run. It explains what happened overnight, what the agent found, what changed, what was verified, and what the human should do next.

Status-first Problems found Lessons learned Validation proof Residual risks

GitHub rendering note

GitHub's repository file browser usually shows committed HTML as source. This page is meant to render when opened locally, served from GitHub Pages, or published by a project preview system. The generated readout itself still belongs in /tmp by default unless the user asks for a committed artifact.

What the readout answers

  • What did the elves do while I was away?
  • What problems did they find?
  • What changed in the code, docs, tests, or process?
  • How do we know the work is safe?
  • What lessons should survive into future runs?
  • What should I review, merge, deploy, or plan next?

Source of truth

The page is synthesized from durable run artifacts and tool checks, not memory.

  • survival guide for current state and Stop Gate
  • .elves-session.json for structured batch and review state
  • execution log for proof and chronology
  • learnings.md for durable lessons
  • gh for live PR and CI status

Example readout shape

1. Final status Am I safe to review?

Branch, PR, head SHA, readiness state, pending checks, merged state if applicable, and the direct recommendation for the human.

2. Problems found What did the run uncover?

Bugs, UX gaps, race conditions, auth boundary risks, artifact issues, review blockers, repeated warnings, and other discoveries that matter beyond the final diff.

3. Lessons learned What should future runs remember?

Durable lessons promoted to learnings.md or .ai-docs/*, plus process adjustments made during entropy checks or final review.

4. Batch timeline What changed, batch by batch?

A compact accordion where each batch records scope, fixes, validation, review result, regression proof, and any residual risk.

5. Validation and review proof How do we know?

Local gates, E2E/browser checks, PR checks, subagent review findings, comments fixed, known non-fatal warnings, and proof that the latest tip was checked.

6. Human next steps What now?

Review focus areas, merge/deploy instructions, follow-up plan ideas, and any live resources or provider caveats the human should know before acting.

HTML by default

Dense accountability needs precise text, links, validation evidence, and residual-risk notes.

Images by request

Generated infographics are useful for presentation, but weaker for audit detail and usage-efficient review.

Temporary unless asked

Readouts live in /tmp by default so product PRs do not accumulate operational artifacts.

This committed page is a proof of concept for the Elves Run Readout feature, not the default output location for real run readouts.
