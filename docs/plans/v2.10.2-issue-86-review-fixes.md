### Batch 2 [B2]: Effort explicitness single authority

low` silently parses yet reports not-explicit — full-run-prepare then substitutes the grok
default `high`, inverting an explicit request. Separately `cmd_implement` (~1797–1805) resolves
effort with a raw adapter string compare while `prepare_full_run` (full_run.py ~3524) normalizes

`prepare_full_run`'s normalized resolution the single default authority: the CLI passes
`effort=None` unless explicit. Persist explicitness so `--resume` without `--effort` keeps an
originally-explicit value. Owned surfaces: `scripts/cobbler_agents.py`,
`scripts/cobbler_runtime/full_run.py` (resolution + persisted state), `tests/`.

**Tasks:**
- [x] Set `allow_abbrev=False` on the operator parser (and subparsers if needed)
- [x] Pass `effort=None` from `cmd_implement` when not explicitly provided; keep normalized
  single-authority resolution in `prepare_full_run`
- [x] Persist the resolved effort and its explicitness in run state so resume honors it
- [x] Regression tests: `--effor low` fails loudly; mixed-case adapter default resolution;
  explicit-medium prepare + flagless resume keeps medium

**Acceptance criteria:**
- [x] [B2-A1] Abbreviated long options are rejected: `--effor low` (and `--eff low`) exit with a

- [x] [B2-A3] A prepare with explicit `--effort medium` followed by a `--resume` without
  `--effort` rebuilds at `medium`, not the flipped default; test covers it
- [x] [B2-A4] Full unittest suite passes with no existing test weakened, skipped, or deleted
