# Plan: v1.15.0 Cobbler

## Mission

Release `v1.15.0 Cobbler`: a user-facing layer that makes Elves feel like a natural chat-native
coordinator for independent agent lenses. Cobbler is the coordinator inside Elves: the user asks
once, Cobbler decides whether to answer directly, bring in specialist elves, or convene a read-only
council, and then returns one fitted recommendation with dissent, risks, and next action.

The release should preserve the useful `v1.14.0` Council mechanics while reorganizing the product
language around Cobbler. Normal use must rely on the host platform the user already has - Claude
Code subagents in Claude Code, Codex subagents in Codex, and direct read-only lens analysis when
subagents are unavailable. OpenRouter or other external provider keys are optional advanced
configuration only.

Tagline to preserve across the release:

> Meet the Cobbler: ask once, get a fitted answer from the elves.

## Product Shape

### Hierarchy

- **Elves:** the overall autonomous work system and skill.
- **Cobbler:** the coordinator inside Elves and the main user-facing name for this experience.
- **elves:** the worker subagents or independent lenses directed by Cobbler.
- **council:** a temporary read-only gathering Cobbler may convene when several perspectives help.
- **fitted answer:** the synthesized output Cobbler returns to the user.
- **provider-backed council:** optional advanced model diversity when configured; not the default.

### Public Wording

Use this as the base public explanation:

```text
Cobbler is the coordinator inside Elves. Ask Cobbler a hard question, and it decides how much help
to bring in: a direct answer, a few specialist elves, or a read-only council of independent lenses.

The user gets the fit, not the chatter: one recommendation, the reason it fits, the strongest
dissent, the risks, and the next move.

Cobbler uses the agent platform you already have. In Claude Code, it can use Claude Code subagents.
In Codex, it can use Codex subagents. When subagents are unavailable, it performs the same read-only
lens analysis directly. Normal Cobbler use requires no OpenRouter or external provider key.
```

### Output Shape

Default Cobbler responses should use this shape unless the context calls for very short prose:

```text
Recommendation
Why this fits
Strongest dissent
Risks
Next move
Confidence
```

Raw role reports stay hidden by default. Show them only for verbose, show-reports, or JSON modes.

### Invocation

- **Claude Code primary:** `/cobbler <task>`
- **Claude Code compatibility:** `/council <task>`, `/ec <task>`, `/elves-council <task>`
- **Codex primary:** `$elves cobbler: <task>` or natural language such as "Ask the Cobbler..."
- **Codex compatibility:** `$elves council: <task>` and natural language references to Council
- **Do not add yet:** `/cobble`; keep the alias set small.

Host-specific honesty matters. Do not describe Codex as having a top-level slash command unless the
surface actually supports one. For Codex, document the reliable skill invocation form and mention
Codex Goals as an optional continuation backend for full Elves runs.

## Scope

### In Scope

- Reframe the existing Council feature as Cobbler-first in `SKILL.md`, `AGENTS.md`, README,
  changelog, config examples, and references.
- Keep `/council`, `/ec`, and `/elves-council` as compatibility aliases that invoke the same
  Cobbler behavior.
- Add or update Claude Code alias skill surfaces so `/cobbler` is real for Claude Code users,
  without overwriting user-owned local skills.
- Clarify Codex usage with `$elves cobbler: ...` and native Codex subagent behavior.
- Update prompt/reference templates so the default answer is a fitted recommendation, not a raw
  report dump.
- Add repo consistency checks and tests that prevent drift across Cobbler, Council compatibility,
  provider policy, version metadata, and forbidden public wording.
- Bump the release version to `1.15.0` and update installed Claude/Codex skill bundles if the
  sync script supports it safely.

### Out of Scope

- Building a hosted slash-command parser, daemon, MCP server, or OpenRouter runtime.
- Making OpenRouter or any external provider required for normal Cobbler use.
- Copying vendor identity, policy, persona, or safety framing from any external prompt.
- Using public wording that markets Cobbler through an external vendor prompt, vendor identity, or
  sloppy-assembly joke.
- Adding `/cobble` as an alias in this release unless a review finds a strong reason.
- Changing the math workflow provider policy except where docs need to distinguish math from
  Cobbler.

## Non-Negotiables

- User experience comes first: the happy path should feel like asking one capable coordinator, not
  operating a provider router.
- Normal Cobbler use must not require OpenRouter, `OPENROUTER_API_KEY`, or any external provider
  key.
- Cobbler must use host-native capability first: Claude Code subagents in Claude Code, Codex
  subagents in Codex, and direct read-only lens analysis when subagents are unavailable.
- Public docs must not advertise or copy external vendor identity, persona, policy, safety framing,
  or external vendor-prompt marketing.
- `SKILL.md` and `AGENTS.md` must move together for behavior changes.
- The agent never merges by default. The PR is for the user to review and merge unless the user
  explicitly opts into merge-on-green or invokes the reviewed-PR landing command. Any opt-in landing
  uses a regular merge commit, never squash or rebase.

## Test Strategy

- **Primary consistency gate:** `python3 scripts/check_repo_consistency.py`
- **Unit tests:** `python3 -m unittest discover -s tests -p 'test_*.py'`
- **Script compile gate:** `python3 -m py_compile scripts/check_repo_consistency.py scripts/install_doctor.py scripts/sync_installed_skills.py scripts/validate_survival_guide.py`
- **JSON validation:** `python3 -m json.tool config.json.example >/dev/null` and
  `python3 -m json.tool .elves-session.json >/dev/null` during the live run
- **Whitespace gate:** `git diff --check`
- **Install/sync check:** `python3 scripts/sync_installed_skills.py --check`
- **Preflight:** `ELVES_SURVIVAL_GUIDE_PATH=docs/elves/survival-guide.md ./scripts/preflight.sh`
- **PR review loop:** after every push, read PR comments, review threads, checks, and bot output;
  fix blockers before continuing.
