# Plan: v1.17.0 Cobbler Harness Loop

## Mission

Upgrade Cobbler so it captures the useful harness engineering behind Claude Fable in spirit, while
staying Elves-native and free of vendor identity, persona, policy text, and safety framing. The
observable outcome is simple: a user reading the docs or invoking Cobbler should see a coordinator
that classifies intent, scans available capabilities, routes agents/tools/skills, passes context
packets, collects evidence, fits one answer, records material run decisions, and reclassifies when
the evidence changes.

## Source Grounding

- Cobbler is inspired by the harness engineering ideas in
  [Claude Fable 5](https://github.com/elder-plinius/CL4R1T4S/blob/main/ANTHROPIC/CLAUDE-FABLE-5.md),
  as extracted by Pliny the Prompter.
- Borrow the harness pattern only: task routing, capability discovery, skill/tool use, state
  rehydration, output selection, and synthesis.
- Do not copy Fable identity, vendor policy, persona, refusal framing, or safety guardrails.

## Scope

### In Scope

- Name and document the Cobbler harness loop across user docs, agent docs, and reference prompts.
- Make the loop concrete enough for Codex and Claude Code users: capability scan, route and medium,
  context packet, execution, evidence collection, fitted answer, record or present, reclassify.
- Keep host honesty: Claude Code has slash alias skills; Codex uses `$elves cobbler: <task>` or
  natural language.
- Keep provider policy unchanged: normal Cobbler needs no OpenRouter or external provider key.
- Add repo consistency checks and tests so the harness-loop language stays aligned.
- Land the PR with a merge commit, then bump and publish the next GitHub release tag.

### Out of Scope

- Building a hosted runtime, parser, CLI, or provider router.
- Requiring OpenRouter, external model keys, or provider-backed council for ordinary use.
- Reworking the math workflow provider policy.
- Replacing the existing Elves loop or adding a separate Cobbler ledger.
- Updating the infographic unless explicitly requested again.
