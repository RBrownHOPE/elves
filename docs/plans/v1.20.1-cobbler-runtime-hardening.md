# Plan: v1.20.1 Cobbler Runtime Hardening and Optional Host Parity

## Mission

Ship a corrective v1.20.1 release that makes Cobbler's external-agent runtime truthful, safe, and
pleasant to use from both Codex and Claude Code. A successful council lane must represent a real
agent/model turn; persistent sessions must resume the exact recorded agent; external writer output
must cross a race-safe audited patch boundary; and setup must preserve user configuration and never
pretend that detection is qualification.

The complete zero-config experience remains host-native. External CLIs, API wrappers, provider keys,
model recommendations, persistent sessions, and external implementation are optional capability
routes. A user who wants to work entirely in Codex, entirely in Claude Code, or in another capable
host must never be nagged, blocked, or silently routed elsewhere.

Done means the repaired runtime passes adversarial tests and real local contract probes, the three
available Claude/Fable, Grok Build, and Sakana/Fugu routes pass a live end-to-end qualification with
their actual models and session evidence, native-only behavior passes with every external route
disabled, Codex and Claude Code documentation/install surfaces agree, PR review and CI are clean,
and v1.20.1 is merged with a regular merge commit, tagged, released on GitHub, and installed locally
for both hosts.

## Why this is a patch release

The v1.20.0 architecture and product direction are worth preserving, but its acceptance evidence was
not strong enough. The repository's 278 tests pass while several tests encode placeholders or fake
commands that cannot work against the installed CLIs. This release corrects truthfulness, safety,
configuration, and host-parity defects without creating a second orchestration system or a broad
provider SDK.

The main coordinator is expected to hold more context and exercise better judgment than any
implementation worker. Every batch contract and worker packet must therefore explain intent,
rationale, existing surfaces to extend, owned and forbidden files, exact behaviors, failure modes,
and acceptance evidence. A worker must not need this chat or model prestige to infer the job.

## Source architecture to preserve

1. **Elves executes:** plans, worktrees, commits, PRs, validation, memory, review, and landing.
2. **Cobbler coordinates:** intent, capabilities, routes, independent evidence, dissent, and one
   fitted answer.
3. **The active host owns truth:** canonical run documents, branch refs, pushes, acceptance,
   synthesis, and release actions.
4. **Providers are routes, not orchestration:** built-in Claude/Grok/Codex presets and custom
   wrappers implement one provider-neutral contract.
5. **Native-only is complete:** no setup, provider key, external binary, or `.elves` mutation is
   required for ordinary Cobbler or Elves use.

Build on these existing surfaces instead of duplicating them:

- `scripts/cobbler_agents.py`: operator CLI and stable JSON boundary.
- `scripts/cobbler_runtime/{schema,config,capabilities,context,adapters,dispatch}.py`: route and
  read-only council pipeline.
- `scripts/cobbler_runtime/{sessions,leases,audit}.py`: exact-session and external-writer boundary.
- `scripts/workspace_guard.py`, `scripts/preflight_worktree.py`, and
  `scripts/elves_landing_check.py`: existing git safety and acceptance concepts.
- `scripts/sync_installed_skills.py` and `scripts/install_doctor.py`: release/install parity.
- `SKILL.md`, `AGENTS.md`, `README.md`, `docs/cobbler.md`, setup/config references, and Claude
  aliases: canonical user behavior.
- Existing standard-library `unittest` fake executable/repository patterns under `tests/`.

### Blocking defects the run must close

**Dispatch and routing**

- Built-in command builders use unsupported `--packet`, `--prompt-file`, or `--session-create`
  combinations for the wrong CLIs.
- `host-native` returns a canned PASS report, so three fake lanes can satisfy quorum without any
  analysis.
- Council construction drops configured executable, requested model, required policy, extra args,
  and fallback chains; parsed fallbacks are not executed.
- A requested model can succeed without authoritative actual-model evidence.
- Timeout kills the direct child only, and second-resolution run IDs can collide.
- Secret-looking environment names are stripped even when a profile explicitly needs a named API
  credential, making generic API wrappers unusable.

**Sessions and writer safety**

- Expected context drift marks rehydration required but also stores the new digest, allowing a later
  resume to activate without proving rehydration.
- Session/lease registries write non-atomically and lease exclusivity is a check-then-write race.
- `.lstrip("./")` path handling can transform `.elves/...` into an allowed `elves/...` path.
- Writer preparation accepts arbitrary session IDs and assumes Grok write capability from version
  or executable presence rather than binding exact qualified evidence.
- Linked-worktree audit watches the worktree git dir but misses shared common-dir config/hooks.
- Export can re-audit without the original evidence, export stale patches, or advance a rejected
  lease; dependent patch series are apply-checked independently against the same base.

**Setup, UX, and release evidence**

- Setup rewrites recognized TOML sections, interpolates untrusted names/values, can leave invalid
  TOML, and echoes raw preferences while claiming credentials were not printed.
- `model_routing.enabled`, session defaults, and sharing policy do not reliably affect resolution.
- `--smoke` does not make a model call; doctor/setup conflate executable detection with auth,
  discovered model, and qualification.
- Read-only commands may create runtime directories merely by constructing a registry.
- Per-project Claude installation omits the two setup aliases.
- The previous run checked mandatory live acceptance boxes despite incomplete live and mirror
  evidence. v1.20.1 must not repeat that bookkeeping error.

## Product contract

### Literal capability states

Use distinct, evidence-backed states:

- `detected`: executable or configured wrapper exists.
- `probed`: a bounded non-inference capability/version/auth probe actually ran.
- `qualified`: the recorded executable identity/version passed the relevant behavioral contract.
- `smoked`: a bounded inference turn actually ran and returned valid structured evidence.
- `unknown`: evidence is absent; it never means qualified and never enables a write route.

Preference files may request capabilities but cannot self-declare qualification. Behavioral evidence
belongs under ignored runtime state and includes executable identity, version, timestamp, contract
version, requested/actual model, and the exact capability proved.

### Minimal provider-neutral wrapper contract

The core resolver understands profiles, role chains, capabilities, argv templates, input/output
contracts, named environment requirements, and evidence. Built-in Claude, Grok, and Codex/Fugu
support is shipped as version-aware presets. A custom local wrapper can participate without source
changes by consuming a redacted packet/prompt and returning a structured role-report envelope that
includes `actual_model` and session metadata when relevant.

This patch does not build a provider marketplace, store credentials, or promise that every API has
a native adapter. A user can wrap Antigravity, Gemini/Google Ultra, OpenRouter, Muse, a local model,
or a future CLI behind the same small contract.

### Host-native truthfulness

A shell script cannot impersonate the current Codex or Claude host. A host-native report counts only
when the active host injects a real subagent/direct-analysis report through the runtime boundary.
Without such an executor/report, the CLI records the lane as unavailable and asks the host to
synthesize; it does not fabricate evidence or mark council quorum verified.

### Configuration semantics

- External routing disabled or absent => every role resolves host-native and launches nothing.
- Role chains preserve profile, executable, requested model, extra args, session mode, required
  policy, and ordered fallbacks end to end.
- `required` blocks only when explicitly recorded in the active Survival Guide/project route and no
  qualified candidate succeeds.
- Session mode and context-sharing policy are separate from whether the local config file may be
  shared.
- Credentials are referenced by environment-variable name or local wrapper, never stored. A profile
  may explicitly grant a minimal named secret environment to its child; values never enter packets,
  argv, logs, JSON, or committed files.
- Legacy v1 config remains readable. Migration is inspect-first and never silently rewrites an
  unmanaged file.

## Scope

### In scope

- Repair read-only dispatch, fallback, model evidence, process lifecycle, and adapter contracts.
- Repair exact session lifecycle, registry persistence, and rehydration proof.
- Make external writer leases and audit/export/import a fail-closed transaction.
- Make setup/config non-destructive, truthful, redacted, and provider-neutral.
- Align Codex and Claude Code skills, aliases, installation paths, docs, and live qualification.
- Bump all version surfaces to v1.20.1 only after behavior and docs are green.
- Merge with a regular merge commit, create/push local `v1.20.1`, publish the GitHub release, and
  synchronize the maintainer's installed Codex and Claude copies.

### Out of scope

- A hosted provider service, provider marketplace, GUI, daemon, or long-running scheduler.
- Automatic credential retrieval/storage or printing secret values.
- Hardcoded rankings of “best” current models; recommendations are capability-based and dated.
- Requiring Grok, Claude, Sakana, OpenRouter, or any API for ordinary public use.
- Giving external workers branch, push, PR, merge, release, or canonical run-memory ownership.
- Rewriting the entire Cobbler/Elves hierarchy or adding a second Council orchestration layer.
- Publishing a package outside the explicitly authorized GitHub release/tag.

### Batch 3: Transactional external writer boundary

**Intent:** External implementation is untrusted input. Exactly one qualified session gets one
lease for one detached worktree and path set; the host imports only a proven, immutable audited patch
series and remains the sole owner of branch history.

**Build on:** Batch 2 session evidence, `leases.py`, `audit.py`, `workspace_guard.py`, git worktree
helpers, and existing lease/audit tests.

**Tasks:**

- [ ] Replace string stripping with canonical repository-relative path validation. Reject absolute,
  empty, `.`, `..`, symlink-escaping, `.git`, `.elves`, `.elves-session.json`, canonical run memory,
  and any path outside a non-empty explicit allowlist.
- [ ] Bind lease preparation to an exact active `SessionRegistry` record, actual model, profile,
  executable fingerprint/version, fresh write-capability canary, context digest, sandbox, worker
  checkout, base head, and allowed path set. Executable presence/version alone never qualifies write.
- [ ] Make exclusivity and state transitions race-safe and atomic across all linked worktrees in the
  same common repository. Use an explicit compare-and-swap lifecycle:
  `prepared -> active -> auditing -> audited_pass -> exported -> apply_checked -> integrated -> closed`;
  `rejected` is terminal. Corrupt/malformed lease or audit records fail closed and remain visible.
- [ ] Capture both worktree-specific git dir and shared `--git-common-dir`, full refs, remotes,
  config, hooks, status/index, processes, and observed commands before and after the turn.
- [ ] Persist an immutable audit result and evidence digest. Export is legal only from the exact
  `audited_pass` state; it cannot silently rerun a weaker audit or advance rejected/active state.
- [ ] Export into a newly created empty run-specific directory, return only patches generated for
  the audited manifest, hash them, and reject stale/unlisted files.
- [ ] Apply-check the complete dependent patch series sequentially in a temporary index/worktree
  without mutating the host checkout. The host may then import, review, test, and author sanitized
  branch commits; the worker never moves refs or pushes.

**Forbidden shortcuts:** No prompt-only safety; no arbitrary session ID; no implicit Grok profile;
no check-then-write lock; no skipping malformed records; no export-time re-audit without original
snapshots; no per-patch checks against an unchanged base; no cherry-pick across the trust boundary.

**Acceptance criteria:**

- [ ] Adversarial path tests include `.elves/secret.json`, `.elves-session.json`, dot segments,
  absolute paths, Unicode/lookalike edge cases, symlink escapes, empty allowlist, and valid nested
  paths; every forbidden case fails closed.
- [ ] Two simultaneous prepare processes yield exactly one live lease; crash/concurrent save tests
  preserve valid state and a diagnosable lock outcome.
- [ ] Arbitrary/unregistered/drifted/wrong-model/stale-canary sessions cannot prepare a writer lease.
- [ ] A linked-worktree mutation to common config/hooks/remotes/refs is detected even when the
  worktree git dir itself is unchanged.
- [ ] Export before audit, after failed audit, with missing evidence, with stale patch files, or from
  the wrong tip is rejected without state promotion.
- [ ] A two-commit patch where commit 2 depends on commit 1 passes sequential apply-check; reordered,
  modified, extra, or manifest-mismatched patches fail.
- [ ] A real, disposable Grok writer canary is attempted only after all preceding tests pass and is
  accepted only if exact session/model/worktree/capability and full audit evidence pass.
- [ ] Full suite remains at least the Batch 2 count with no new skips; deterministic gates pass.

**Docs likely touched:** external writer lease/audit contract, operator workflow, learnings if a new
stable CLI/worktree fact is discovered.

**Risk:** Git's split worktree/common-dir topology and cross-process locking are subtle. Prefer a
small explicit state machine and mechanical tests over clever abstractions.
