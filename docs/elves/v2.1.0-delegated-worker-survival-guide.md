## Non-Negotiables

- Grok may edit/test/commit/push only the feature branch; it may not merge, tag, force-push, change
  the base branch, open another PR, deliberately inspect/edit credentials through worker tools, or
  edit host-owned run-control fields. Grok's native auth manager may refresh only the explicitly
  granted canonical OAuth file.
- Driver activity after launch is bounded monitoring only until a wake condition or final readiness.
- Never expose secrets. Bootstrap the current Grok process with an explicit minimal environment and
  redirect its full transcript to a private ignored artifact.
- Never use destructive Git commands. An unexpected non-descendant branch move is a collision and
  stops the worker path.
- Legitimate behavior-driven test changes are permitted with preserved/improved coverage and clear
  evidence; deleting/skipping/weakening tests solely to pass is forbidden.
- Preserve host-native operation and Claude Code/Codex parity.
- Merge only via `gh pr merge --merge` after the final readiness review and all PR surfaces are clean.
