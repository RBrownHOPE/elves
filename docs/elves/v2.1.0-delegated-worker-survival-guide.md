- Work as hard as you can through final readiness and landing. Do not be lazy.
- Do not accept the minimum acceptable change or first green signal when a planned criterion,

## Forbidden Stop Reasons

- A checkpoint, commit, or push succeeded.
- A batch completed while later batches remain.
- CI is green but acceptance, review, or landing remains.
- A PR exists or the worker produced a useful summary.
- The user is silent, the run feels large, or the driver has reached a convenient boundary.

  evidence; deleting/skipping/weakening tests solely to pass is forbidden.
