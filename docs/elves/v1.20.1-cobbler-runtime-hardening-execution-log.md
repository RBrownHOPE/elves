### Parallel planning lenses

Three independent read-only lenses inspected distinct source areas; no lens edited files.

1. **Dispatch/adapter lens — Chandrasekhar**
   - Replace each built-in with explicit argv/stdin/decoder contracts and fail unknown versions into
     fallback rather than guessing.
   - Preserve the full resolved route through ordered attempts.
   - Delete canned host-native evidence; standalone native mode may complete directly but cannot be
     called council-verified without a real injected host execution identity.
   - Derive actual model from adapter machine metadata, not model-authored JSON.
   - Run fallbacks sequentially within a logical lane while independent lanes remain parallel.
   - Use process groups, UUID-backed artifact roots, and per-profile named-secret allowlisting.
   - Strongest dissent: a CLI without trustworthy model metadata may still offer unpinned optional
     advice but cannot satisfy an exact-model gate.

2. **Session/writer lens — Beauvoir**
   - Use one common-repository durable store with atomic restricted writes, registry locking,
     collision-safe IDs, revisions/CAS transitions, and malformed-record fail-closed behavior.
   - Keep rehydration required sticky with pending digest/HEAD plus nonce; only matching actual
     acknowledgment adopts pending state.
   - Derive exact resume from the registry and versioned adapter contract; CWD is subprocess metadata,
     not unsupported argv.
   - Bind lease identity immutably and make one lease win across all linked worktrees.
   - Use an explicit linear audit/export/apply/integrate state machine; capture common-dir plus
     worktree git state; hash an immutable patch manifest and cumulatively apply-check it while
     leaving host state byte-identical.

3. **Setup/host/release lens — Maxwell**
   - Keep v1.20.1 a backward-compatible truthfulness/safety patch, not a provider platform.
   - Separate detected, probed, qualified, and smoked states; unknown is not qualification.
   - Keep core routing provider-neutral through a small JSON/stdio-style wrapper contract; built-ins
     are compatibility presets.
   - Make setup inspect-first with explicit apply/probe/real-smoke operations and atomic digest-
     checked persistence that preserves user bytes or refuses safely.
   - Normalize enabled/session/context/config-sharing semantics and read v1 without silent rewrite.
   - Align global/project Codex/Claude installs, all seven Claude aliases, temporary install tests,
     README/CHANGELOG/references, and post-merge tag/release/local sync.
